1 - HTML Markup

There are some issues with this site’s HTML markup. The first major issue is how a div element is used as the child of a span element eight times. This is an error because a block-level element (like a div) can contain both inline and other block-level elements, while an inline element (like a span) can only contain other inline elements. Another issue is the usage of square brackets in the href attribute of an a element, which the W3 Validator states is not allowed. According to the Internet Engineering Task Force (IETF), the square bracket characters are considered unsafe and should be encoded within URLs. A third issue is the repeated usage of the itemprop attribute without specifying what types these properties belong to. It is especially concerning that the W3 Validator encountered this error 12 times.

This site’s HTML also contains several milder issues. For example, 10 different script tags contain the type attribute and the value "text/javascript". In HTML5, the default value for the type attribute is "text/javascript", which makes its many inclusions within this site’s HTML unnecessary. In addition, seven different sections in the markup do not contain any headings, which hurts the accessibility of this website.

To improve this site’s HTML markup, I would first change the span elements containing divs to div elements and revise the URL with square brackets to contain the encoded versions of square brackets. Next, I would use the itemtype attribute to describe what types the itemprop attributes belong to. Finally, I would eliminate the 10 redundancies of type="type/javascript" and add appropriate headings to the seven sections in the HTML. This would ensure the browser can download the HTML and render the page efficiently.

2 - Search Engine Optimization

The most serious issue with this site’s search engine optimization (SEO) is its page speed. Page speed is important because not only is it factored into the page’s rankings in search engine results, but faster sites also improve the user experience and reduce the likelihood they will leave prematurely. One aspect slowing down this site’s loading speed is the lack of browser caching across all resources on the site. Caching is useful because it allows a browser to load a previously visited page faster by not having to re-download the same files from the page’s server. According to varvy.com, browser caching is not enabled for this domain or for third-party resources. A second factor in the site’s slower loading speed is its CSS files. When a browser accesses this site, it must retrieve and download five CSS files, three of which are larger than 90 kilobytes. There are also 36 instances where inline styling is used, which takes up 1,383 bytes. As a result, the browser wastes time downloading and rendering the CSS associated with this site. Another reason for the site’s increased loading time is its JavaScript. When a browser accesses this site, it must download 14 JavaScript files, which have a combined size of 4,244 kilobytes. Additionally, one of these JavaScript files is render blocking, which means no other resources can load until this file is finished. As a result, there is a delay in the time required for the browser to create and display the page. The last major issue is a lack of image optimization across 31 different images on the site. According to varvy.com, optimizing all these images would save more than 2000 kibibytes. In other words, optimizing these images would substantially increase the site’s loading speed.

To improve this site’s SEO, I would first determine which non-cached elements are updated least frequently, and thus would be the best candidates to enable caching for. Next, I would recommend combining the five CSS and 14 JavaScript files into one or two files each, as well as reducing their overall size. This would greatly increase loading speeds by decreasing the number of retrievals from the server, the downloading times, and the rendering times. Also, I would advise examining the render blocking JavaScript file to see if it can be delayed, allowing more important elements to render quicker on the page. Finally, I would suggest optimizing as many of the 31 images as possible, which would allow a browser to download them faster and in turn load the site faster. After implementing these changes, the site’s loading speed should increase, which in turn should improve the site’s search engine optimization.
